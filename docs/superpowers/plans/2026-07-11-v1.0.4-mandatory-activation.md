# v1.0.4 Mandatory Activation Recovery Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ship a verified v1.0.4 replacement whose supported CLI, dashboard, and native desktop paths require live API-backed activation before useful review or provider work, then clean predecessor distribution surfaces without breaking the install/update lane.

**Architecture:** A locked production-policy module converts legacy user config into an immutable canonical policy. A shared live admission service is consumed before every useful-work seam; unsigned cache data remains diagnostic only. CLI, dashboard, and Desktop share one activation contract, while exact-head release gates require real activate/validate/deactivate and no-bypass evidence before workflow-owned publication and cleanup.

**Tech Stack:** TypeScript/Node 22, Vitest, Swift 6/Swift Testing, SwiftUI, macOS Keychain, local HTTP dashboard, GitHub Actions, npm registry, GitHub Releases/REST API.

## Global Constraints

- Canonical API is exactly `https://neondiff-license.fly.dev` in the supported distribution.
- Public, private, internal, and unknown repository work all require active entitlement.
- v1.0.4 grants zero offline cache authority; every useful-work admission validates live.
- Tests use typed dependency injection, never an environment variable or hidden production bypass.
- License keys enter through bounded stdin/native secure input and never appear in argv, logs, issues, screenshots, evidence, or config JSON.
- Legacy v1.0.0–v1.0.3 config loads and can activate, but weakening fields cannot affect the effective policy.
- Do not publish or delete predecessors until exact-head CI, independent reviews, install/upgrade/activation/no-bypass proof, and replacement convergence pass.
- Do not claim source-build prevention, signed/notarized Mac distribution, Sparkle maturity, browser/native parity, GA, or v1.1 completion.
