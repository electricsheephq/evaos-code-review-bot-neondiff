## Purpose

Close the sticky-vs-cold eval comparison lane for `#85` and ship the runtime
safety fixes found while reviewing the bot's own PR. The eval runner compares
paired cold and sticky review packets without making public calibrated-quality
claims, and the scheduler fixes prevent provider-cooldown retries from
misclassifying queue/session state under ZCode provider churn.

## Changes

- Add `eval-sticky-vs-cold` support with paired cold/sticky packet validation,
  comparison gates, summary JSON, markdown reports, and fixture coverage.
- Keep public confidence output conservative: sticky-vs-cold results remain
  advisory unless future batch evidence satisfies the explicit runtime-safe
  thresholds.
- Validate both cold and sticky packet threshold policies before writing
  output artifacts.
- Count negative-control evidence only when explicitly marked negative controls
  are clean.
- Add fail-closed scheduler mapping for missing processed rows so queue jobs
  stay retryable instead of being marked posted without a review URL.
- Keep RepoSticky reviewer session jobs assigned when a processed-head retry
  has no processed row.
- Scope command-recorded retry settlement to manual-command queue jobs so an
  explain/finishing-touch command cannot retire unrelated automatic retry work.
- Preserve provider cooldown retries as visible durable queue state when ZCode
  returns transient overload.
