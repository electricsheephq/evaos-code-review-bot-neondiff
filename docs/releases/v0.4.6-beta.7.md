## Purpose

Fix the retry and failed-head cleanup path that made active agents look stuck
after a ZCode/provider failure. The release keeps destructive cleanup
dry-run-first, repairs stale failed durable queue jobs when a retry has already
posted the review, and makes retry status comments head-specific so other
agents do not wait on the wrong PR head marker.

## Changes

- Make `retire-failed` default to dry-run unless an operator explicitly passes
  `--dry-run false`.
- Expand `retire-failed` dry-run evidence with normalized reason,
  `queueJobsToRetire`, redacted current-row previews, and
  `retiredErrorPreview` using the same template as live state mutation.
- Redact structured retire-failed JSON before serialization so credential URLs
  cannot corrupt parseable operator evidence.
- Let `retry-failed` repair failed durable queue jobs when the processed review
  row is already terminal `posted`, without rerunning ZCode or duplicating the
  review.
- Sync retry status comments against the requested retry head SHA, not the live
  PR head SHA, for stale/closed retry repairs.
- Map capacity-limited `retry-failed` outcomes to sticky
  `provider_deferred`, so agents know the retry remains queued instead of
  treating an old `failed` status as terminal.
