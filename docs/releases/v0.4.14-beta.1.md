## Purpose

Promote the first public GitHub App onboarding slice for NeonDiff. This release
adds a GitHub-only readiness command for App install verification and rewrites
the setup docs around selected-repository install, uninstall, troubleshooting,
license boundaries, and public-safe evidence.

## Included Changes

- Adds `neondiff doctor github`, which verifies GitHub App credential presence,
  App installation visibility, and enabled-repo read access without running
  ZCode, calling a model provider, posting comments, or printing secrets.
- Fails locally when no GitHub App credentials or fallback read token are
  configured, instead of making unauthenticated GitHub requests.
- Reworks `docs/github-app-setup.md` into the canonical public install and
  onboarding guide.
- Links the GitHub App onboarding path from `README.md` and `docs/SETUP.md`.
- Records the selected-repo install path, permission rationale, uninstall path,
  first dry-run review path, public/private license boundary, and evidence
  packet requirements.
- Keeps issue enrichment explicitly separate from PR review through its own
  allowlist and per-repo throttles.

## Caveats

- This release does not close #106 by itself. Remaining acceptance still needs a
  fresh external App install smoke, private-repo missing-license fail-closed
  evidence, a permissions snapshot, and a first App-authored review URL from
  that external install path.
- This release does not publish a GitHub Marketplace listing.
- This release does not expand GitHub App permissions or monitored repo
  allowlists.
- This release does not enable issue enrichment on PR review repos.
