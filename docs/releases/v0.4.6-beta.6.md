## Purpose

Turn the live launchd TLS containment fix into a durable beta gate. The previous
live daemon was running without Node's macOS system trust store option, which
caused GitHub App installation fetches to fail from launchd while interactive
CLI commands worked. This release makes `release:status` verify the loaded
LaunchAgent environment includes `NODE_OPTIONS=--use-system-ca`.

## Changes

- Add a `launchd_node_system_ca` release-status gate.
- Parse loaded launchd `NODE_OPTIONS` from `launchctl print` output with
  quoted-value handling, exact-token matching, and line-scoped environment
  extraction.
- Fail closed when launchd cannot prove the system CA option is present.
- Document the LaunchAgent `EnvironmentVariables` snippet with
  `NODE_OPTIONS=--use-system-ca`.

  `NODE_OPTIONS=--use-system-ca` before this release. This release makes that
  requirement visible to the release gate and docs.

  process startup; the gate therefore requires both launchd running and the
  loaded environment option.
