## Purpose

Promote the issue-enrichment self-trigger fix so manual and live issue
enrichment reruns do not repeatedly update the same sticky comment just because
the bot's own comment advanced GitHub `issue.updated_at`.

This is a local live-worker beta release. It does not publish a new npm package
or replace the public `neondiff@0.4.24-beta.1` package release.

## Included Changes

- Stores structured issue-enrichment body hashes on enrichment comments and in
  `issue_enrichment_records`.
- Uses body hashes to skip unchanged `posted` issue-enrichment records even
  when GitHub `updated_at` is noisy from bot sticky-comment writes.
- Preserves dry-run-to-live behavior so dry-run rows do not suppress the first
  live App-authored comment.
- Preserves changed-body re-post behavior when the rendered enrichment content
  actually changes.
- Adds SQLite migration, state-layer validation, legacy-row, and live cycle
  coverage for body-hash persistence.
