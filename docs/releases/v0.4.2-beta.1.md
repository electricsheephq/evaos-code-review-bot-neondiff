## Purpose

Ship the first eval/calibration substrate for the CodeRabbit-class reviewer
roadmap. The bot can now produce offline eval scorecards with calibration
reports and a release-level promotion decision that keeps public confidence
percentages disabled until the evidence thresholds are actually met.

## Changes

- Add a public confidence policy requiring all of:
  - Wilson lower bound at least `0.95`.
  - At least `100` labeled findings.
  - At least `30` P0/P1 labels.
  - At least `10` negative-control scenarios.
- Add calibration reports with reliability bins, Wilson lower bounds, and a
  public display policy that remains `uncalibrated` below threshold.
- Add a suite-level promotion decision artifact:
  `promotion-decision.md`.
- Add `suite-summary.json` to `eval-suite` root output.
- Guard `eval-suite --output-root` before root artifact writes so eval outputs
  cannot be written inside the repository checkout.
- Keep raw Wilson values for promotion gating so display rounding cannot make a
  borderline bin pass.
- Document the eval artifacts and the public confidence proof boundary.
