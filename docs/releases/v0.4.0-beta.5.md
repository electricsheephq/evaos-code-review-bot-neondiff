## Purpose

Make coverage audit and operator status accurately distinguish unprocessed PR
heads from heads already covered by durable review queue work.

This fixes a live rollout gap where eligible open PR heads could appear
unprocessed even though the scheduler had already queued, leased, running, or
provider-deferred review work for the exact current head.

## Changes

- Add a `queued` coverage-audit bucket for active durable queue coverage.
- Require queue coverage to match exact `{repo, pull_number, head_sha}`.
- Require automatic queue rows to match the current base SHA, while preserving
  legacy automatic rows with `base_sha = NULL`.
- Require `leased` and `running` rows to be non-expired before they cover a
  current head.
- Route `provider_deferred` queue rows into the provider-deferred bucket instead
  of generic queued coverage.
- Preserve queue `createdAt`, `updatedAt`, `nextEligibleAt`, source, lane, and
  priority in operator queue/dashboard/explain surfaces.
- Surface durable provider-deferred retries before returning older skipped
  provider-cooldown processed rows.
