## Purpose

Fix a live release-health regression where provider-deferred queue rows whose
`nextEligibleAt` had expired could keep `release:status` and runtime inventory
red even when the scheduler could not safely act yet because provider capacity
was full.

## Changes

- Add explicit provider-deferred readiness counts to review budget status:
  total, retryable, ready-to-retry, cooldown waits, and capacity waits.
- Make `release:status` fail provider-deferred backlog only when budget says a
  provider-deferred job is actually ready to retry.
- Make operator status and runtime inventory use the same ready-to-retry count
  instead of raw timestamp-only retryable counts.
- Preserve raw retryable provider-deferred counts in queue summaries so the
  operator can still see deferred pressure without treating all of it as
  release-blocking debt.
