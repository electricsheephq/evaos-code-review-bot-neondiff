## Purpose

Add default-off GitHub related-context packets so reviewer prompts can include
explicitly linked issue/PR context without letting that context become review
truth.

This closes the GitHub enrichment half of the repo-memory/GitNexus context
sprint: current diff, checkout files, and GitHub PR metadata remain
authoritative, while related issue/PR metadata is bounded, redacted, quoted as
untrusted data, and advisory only.

## Changes

- Add `githubRelatedContext` config with default-off activation, byte/token
  caps, per-request timeout, and disabled cross-repo refs by default.
- Add a `build-github-related-context-packet` operator CLI command that reads
  explicit PR title/body references and writes JSON/Markdown evidence without
  posting, calling ZCode, applying labels/reviewers, or searching beyond
  explicit refs.
- Add App-token backed related issue/PR reads.
- Add worker/prompt integration behind the default-off flag.
- Quote fetched titles, URLs, authors, labels, milestones, and body excerpts as
  untrusted prompt data.
- Ignore token-shaped repo references and fail closed if rendered packet
  markdown still contains secret-like text.
- Add regression coverage for timeout degradation, rate-limit early stop,
  config/builder validation parity, secret redaction, and runtime aborting
  GitHub reads.
