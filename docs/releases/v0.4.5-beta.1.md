## Purpose

Ship the issue-enrichment safety gate required before any live issue-comment
rollout: global caps across repos, a single-worker lease, operator visibility,
and explicit force semantics for clearing active issue-enrichment leases.

This release also fixes a ZCode policy-cleanup bug found during PR validation:
if ZCode removed the temporary `.zcode` directory while failing with provider
overload, cleanup could mask the real provider error as a local `ENOENT`.

## Changes

- Add global issue-enrichment caps for max issues/comments per cycle.
- Add a SQLite-backed issue-enrichment worker lease with TTL.
- Keep issue enrichment live-disabled by default with a separate empty allowlist.
- Preserve activation watermark behavior so newly allowlisted repos do not sweep
  historical open issues unless explicitly configured.
- Add `clear-issue-enrichment-leases` operator command with dry-run and
  confirmation gates.
- Require `--force-active true` before clearing non-expired issue-enrichment
  leases in non-dry-run mode.
- Surface active vs expired lease matches in the command JSON output.
- Restore existing `.zcode/config.json` safely even when the ZCode process
  removes the `.zcode` directory during shutdown/failure.
