## Purpose

Add durable, machine-readable markers to generated walkthrough comments so the
bot can safely maintain one sticky walkthrough surface per PR while still
recording the exact reviewed head, event verdict, and body hash.

This release is a schema and rendering hardening slice for `#20`. It does not
enable a new live posting lane by itself: `walkthrough.postIssueComment` remains
controlled by config and the active live config is unchanged from
`v0.3.9-beta.1`.

## Changes

- Replace the previous free-form walkthrough marker with a stable PR-scoped
  marker:
  - `repo=<owner/repo>`
  - `pr=<number>`
- Add a versioned head-specific walkthrough state marker containing:
  - schema version
  - repo and PR number
  - reviewed head SHA
  - review verdict
  - SHA-256 hash of the redacted visible walkthrough body
- Redact the visible walkthrough body before hashing and composing bot-owned
  hidden markers.
- Strip user-authored HTML comments from rendered PR title text before it enters
  the public walkthrough body, preventing marker-injection through PR metadata.
- Update GitHub App read/post tests and walkthrough rendering tests for the new
  marker contract.
