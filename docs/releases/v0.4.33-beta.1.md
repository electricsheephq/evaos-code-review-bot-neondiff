## Purpose

Promote the post-`v0.4.32-beta.1` batch as a named live-worker/source beta
rather than leaving launchd on untagged `main`.

This release tightens at-most-one-review-per-head behavior for manual and
daemon races, improves retry-degraded finding provenance, adds opt-in P0/P1
self-consistency re-check support, records post-merge outcome labels for the
calibration loop, adds batch calibration aggregation, documents
ranking/scoring configuration, and refreshes the hosted BYOK provider smoke
safety gate.

## Included Changes

- Add integration coverage for risk-weighted queue priority resolution.
- Add an atomic per-head review claim so manual exact-head reviews and daemon
  reviews cannot both post for the same `{repo, pr, head_sha}`.
- Add robust false-positive learning with coarse fallback fingerprints and
  human-confirmed severity override handling.
- Add retry-degraded provenance, optional quieter-only confidence penalty, and
  bounded schema-failure retry behavior.
- Add opt-in P0/P1 self-consistency re-check support with quieter-only merge
  semantics.
- Document the ranking/scoring config surface for operators and future
  reviewers.
- Add post-merge outcome observer and finding-outcome label storage as
  calibration-loop part A.
- Add batch calibration aggregation over finding-outcome labels as
  calibration-loop part B.
- Refresh hosted BYOK provider smoke checks so remote smoke remains disabled by
  default and requires explicit provider selection, explicit remote-smoke opt
  in, HTTPS, env-key-backed auth, present key env, safe DNS, pinned lookup, no
  credential-bearing URLs, bounded response reads, redirect fail-closed
  behavior, and redacted/classified diagnostics.

## Caveats

- This release does not publish a new npm package; the public npm package
  remains `neondiff@0.4.30-beta.1`.
- Hosted BYOK remote smoke does not make NeonDiff a hosted proxy or SaaS. It
  only proves an explicit provider-direct smoke path selected by the user.
- Remote smoke still does not send PR diffs, private prompts, fixture review
  payloads, or GitHub data to hosted providers.
- Outcome labels and self-consistency checks are inputs to future calibration;
  this release does not claim CodeRabbit parity, calibrated 95% confidence, or
  production all-org readiness.
- GitNexus remains advisory/read-only where enabled; this release does not
  expand GitNexus runtime permissions or target-repo mutation.
- This release does not expand monitored repos, GitHub App permissions, native
  ZCode skills, MCP tools, memory, shell tools, auto-approval behavior, or
  protected-branch write behavior.
