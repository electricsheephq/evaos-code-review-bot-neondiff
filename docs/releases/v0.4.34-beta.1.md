## Purpose

Promote the root changelog and GA versioning policy work as a named source
beta, so public documentation and release governance do not drift ahead of the
live operator checkout without a recorded release boundary.

This release is documentation and governance only. It does not change live
review behavior, provider execution, GitHub App permissions, or the public npm
package.

## Included Changes

- Add a root `CHANGELOG.md` that summarizes the public beta line and links the
  versioned release notes.
- Document the GA versioning line and npm dist-tag policy for stable and beta
  releases.
- Update the beta release runbook so changelog updates are part of release
  discipline.
- Expand the superseded `v1.0.0-beta.1` note so readers understand why the
  public beta returned to the `v0.4.x` train.

## Caveats

- This release does not publish a new npm package; the public npm package
  remains `neondiff@0.4.30-beta.1`.
- This release does not claim GA readiness. It closes only a documentation and
  versioning slice of #319/#235.
- This release does not change runtime review behavior, provider selection,
  model quality, hosted BYOK behavior, monitored repos, GitHub App permissions,
  native ZCode skills, MCP tools, memory, shell tools, or target-repo mutation.
- CodeQL for the direct push may still run separately; CI for the merged PR was
  green before this release packet.
