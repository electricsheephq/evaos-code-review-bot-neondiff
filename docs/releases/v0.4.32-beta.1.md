## Purpose

Promote the issue-enrichment planner packet as a named live-worker beta rather
than leaving launchd on untagged `main`.

This release adds a conservative Agent-start planning packet for issue
enrichment comments. It summarizes related context, issue shape, product fit,
build/borrow/buy framing, source taxonomy, acceptance/proof prompts, known
traps, and non-goals while keeping issue enrichment on its separate allowlist
and throttle surface.

The release also includes documentation-only source updates for provider
compatibility resources, agent runtime adapter discovery, and the GitNexus
read-only adapter research/eval plan.

## Included Changes

- Add issue-enrichment planner packet rendering with:
  - same-repo related context bounded by `maxRelatedRefs`;
  - conservative external-research gating for product, UX, library-choice,
    build-vs-buy, and market-positioning issue classes;
  - source taxonomy that distinguishes internal repo/GitNexus context,
    same-repo issues/PRs, allowlisted cross-repo GitHub, external OSS examples,
    library/API docs, current-market examples, and internal MCP docs;
  - safety non-goals that always preserve the no-bulk-enrichment and
    no-uncited-research boundaries;
  - final assembled-body redaction before any sticky comment is posted.
- Add focused planner tests for low suggestion caps, related-context cap
  consistency, whitespace-safe problem-shape excerpts, and routine
  GitHub/release/API issue text that must not trigger external research.
- Document provider compatibility resources and public hosted/local provider
  boundaries.
- Document agent runtime adapter discovery constraints and GitNexus read-only
  adapter research/eval notes.

## Caveats

- This release does not expand the issue-enrichment allowlist, active issue
  posting scope, monitored PR repos, GitHub App permissions, provider choices,
  native ZCode skills, MCP tools, memory, shell tools, auto-approval behavior,
  or target-repo mutation.
- Issue enrichment remains a separate lane from PR review allowlisting and must
  remain separately throttled before broader active rollout.
- GitNexus remains advisory/read-only where enabled; this release does not
  change live GitNexus runtime defaults.
- This release does not claim CodeRabbit parity, calibrated confidence, model
  quality parity, or production all-org readiness.
- The public npm package remains `neondiff@0.4.30-beta.1`; this release tags
  the live local worker source SHA only.
- The public website still requires the separate Lovable Production Publish
  lane tracked outside this release.
