## Purpose

Promote the source-only prerelease publish guard so GitHub release tags that do
not change `package.json` no longer fail the npm publish workflow. Source-only
tags must be explicitly listed in `docs/public-release-manifest.json` before
the tag is created.

This release does not publish a new npm package, widen runtime permissions,
change monitored repos, or change review behavior. It prevents source/local
worker beta tags from leaving misleading failed npm workflow runs when the
public package artifact remains intentionally held.

## Included Changes

- Classify npm publish attempts before install/build/pack/publish work begins.
- Allow source-only prerelease tags to skip npm successfully only when the tag
  is listed in `packageArtifact.skippedPublicPackageVersions`.
- Keep manual npm workflow dispatch strict: manual tags that do not match
  `package.json` version still fail closed.
- Gate every downstream publish step on `should_publish == 'true'`.
- Record `v0.4.31-beta.1` through `v0.4.36-beta.1` as skipped source-only
  releases while `neondiff@0.4.30-beta.1` remains the current public package.
