## Purpose

Promote the Sprint C GitNexus advisory context packet slice. This release adds
a default-off GitNexus packet provider so future review prompts can receive
bounded, redacted, freshness-aware related-code context without indexing during
review runs or widening ZCode tools.

This closes `#82`. It does not enable GitNexus context in live config, index
repositories, change App permissions, enable native ZCode skills/MCP/memory, or
claim review-quality uplift.

## Changes

- Add `gitnexusContext` config with conservative disabled-by-default controls.
- Add `src/gitnexus-context.ts` for read-only packet generation, alias
  resolution, GitNexus index freshness detection, changed-file and exported
  symbol hints, related context queries, packet hashing, budget enforcement,
  generated-path omission, and secret fail-closed handling.
- Add `build-gitnexus-context-packet` to the operator CLI for dry-run packet
  evidence without posting, ZCode execution, project tests, or GitNexus
  indexing.
- Integrate GitNexus packets into worker evidence and ZCode prompt construction
  as advisory context only when explicitly enabled and safely built.
- Add focused tests for missing/stale GitNexus degraded mode, configured alias
  matching, exported-symbol extraction, query budgeting, generated-file
  omission, redaction failure, worker evidence, and prompt integration.
- Document the new operator command and safety boundary in `docs/operator-cli.md`.
