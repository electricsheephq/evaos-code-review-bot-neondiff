## Purpose

Promote the first RepoSticky reviewer-session state layer for the v0.2
CodeRabbit-class reviewer roadmap. This patch adds durable session and job
accounting so later queue, inventory, memory, and GitNexus work can reuse
repo-scoped reviewer context without weakening duplicate suppression.

## Changes

- Add disabled-by-default `reviewerSessions` config with TTL and head-count
  limits.
- Add `reviewer_sessions` and `reviewer_session_jobs` SQLite tables.
- Add transaction-safe session assignment, exact-head job dedupe, lifecycle
  transitions, foreign-key enforcement, and dead-worker session avoidance.
- Use `draining` for limit-reached in-flight sessions until terminal jobs
  complete, then expire them.
- Add `doctor` and `release:status` visibility, including repo-scoped reviewer
  session counts.
- Keep ZCode invocation, prompt policy, monitored repos, App permissions,
  command handling, native skills, MCP, and posting policy unchanged.

## Runtime Caveats

- `reviewerSessions.enabled` remains `false` in live config until #83 wires the
  bounded provider-aware queue.
- Manual command attempt identity is intentionally deferred to #83. The #80
  state layer preserves automatic `{repo, pr, head_sha}` dedupe.
- This release changes release-health/state interpretation. It does not expand
  live review concurrency or monitored repos.
- Post-release gates must run `release:status`, `coverage-audit`, and
  `provider-cooldowns --expired-only true` before the release is called green.
