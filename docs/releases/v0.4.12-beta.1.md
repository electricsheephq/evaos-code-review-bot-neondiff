## Purpose

Promote the first NeonDiff Desktop visual shell slice: custom local-first macOS
chrome, an operator-facing dark theme, and a dormant Sparkle update scaffold.
This release keeps desktop distribution claims narrow: no signing,
notarization, DMG, appcast, real update keys, or public installer is included.

## Included Changes

- Adds NeonDiff Desktop custom chrome, dark operator theme, angular panels,
  sidebar polish, settings affordances, and native monospaced display typography.
- Adds a dormant Sparkle dependency and update controller path with explicit
  development-scope guards for feed URL and public key activation.
- Extends the desktop build script to copy Sparkle.framework into
  `Contents/Frameworks` and validate the app binary rpath during bundle checks.
- Bundles Sparkle's license and external dependency notices under the desktop
  Legal resources and points `THIRD_PARTY_NOTICES.md` at that bundled notice.
- Removes the bundled SAIBA-45 font and GPL font license resources from the
  desktop target; the shipped UI uses native system fonts.

## Required Gates

- `swift build`
- `swift run NeonDiffDesktopCoreSmoke`
- `swift package clean && rm -rf dist && ./script/build_and_run.sh --bundle-check`
- `git diff --check`
- PR checks green and current-head actionable review threads resolved.
- Post-merge release-status, coverage-audit, provider-cooldowns, launchd
  restart, and evidence packet before tagging.

## Caveats

- This is a development MVP for local desktop shell behavior, not a signed or
  notarized public desktop release.
- Sparkle remains dormant unless both feed URL and public EdDSA key are
  configured; no production appcast or updater key is shipped.
- No auto-update entitlement enforcement or license-aware update channel is
  activated in this release.
