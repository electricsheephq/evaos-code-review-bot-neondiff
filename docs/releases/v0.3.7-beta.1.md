## Purpose

Promote Sprint 1 deterministic review safety foundations for the live beta
reviewer. The release turns model findings into a more auditable review plan:
findings are validated against current diff lines, categorized through a
deterministic taxonomy, routed through one review-event gate, and paired with
changed-surface validation/proof recommendations in the walkthrough.

This release does not expand monitored repos, GitHub App permissions, ZCode
tools, native skills, MCP, memory packets, GitNexus context, auto-merge,
approvals, or repo mutation.

## Changes

- Add a deterministic regression taxonomy covering data loss, auth, security
  boundary, CI/build, Unity scene/prefab, migration, API compatibility, release
  regression, proof gap, docs-only, dependency, runtime correctness, and unknown
  findings.
- Keep `severity` authoritative for `REQUEST_CHANGES`: any surviving P0/P1
  finding in the current taxonomy remains eligible to block.
- Treat model-supplied `category` as an optional hint. Unsupported categories
  are ignored and inferred deterministically instead of dropping the finding.
- Centralize review gating in `applyDeterministicReviewGate`, including diff
  coordinate validation, secret/cap drops, category counts, drop reason counts,
  and final event selection.
- Add changed-surface validation and proof recommendations for docs, Unity,
  TypeScript/web, CI/release, and the bot's own runtime surfaces.
- Write redacted validation/proof/gate artifacts into each review evidence
  directory.
- Extend walkthrough comments with risk taxonomy and validation/proof sections.

## Runtime Caveats

- Validation/proof recommendations are advisory in this beta. They appear in
  walkthrough comments and evidence, but they do not run project tests or
  execute arbitrary PR code.
- Changed-surface validation intentionally uses the full changed file list so
  proof expectations can mention files excluded from the inline-review prompt
  by repo policy. Inline comments still target validated current RIGHT-side diff
  lines only.
- Category labels are deterministic routing hints. They do not claim calibrated
  confidence or CodeRabbit parity.
- `why` may report a head as pending while the durable scheduler has an active
  queue job. Use `runtime-inventory` when diagnosing scheduler coverage.
