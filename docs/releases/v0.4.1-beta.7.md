## Purpose

Promote the merged `run-once` structured summary CLI behavior so live operators
no longer have to infer scoped dry-run/live outcomes from secondary commands.
This release makes the current live checkout expose parseable JSON summaries for
`run-once`, including scoped PR metadata, skip buckets, failed review counts,
and nonzero failure status after printing the report.

## Changes

- Print structured JSON for `run-once` dry-run and live invocations.
- Include scoped `repo`, `pullNumber`, `headSha`, and PR URL when available.
- Preserve stdout-first operator behavior for successful and failed review
  attempts.
- Return exit code `1` when a review run records one or more failed reviews.
- Emit structured JSON error envelopes for scoped execution failures before
  returning exit code `1`.
- Redact secret-like PR-controlled strings before serializing operator JSON.
- Document that exit code `1` can represent partial per-PR failures, not only a
  total command failure.
- Remove `npx tsx` shell-outs from the focused run-once CLI tests to reduce CI
  flake and cold-start cost.
