## Purpose

Promote Sprint A worker-budget visibility for the live beta reviewer. The
release adds deterministic operator-facing budget projection across global,
provider, organization, repository, and manual-reserve capacity without changing
the live concurrency default.

This release does not expand monitored repos, GitHub App permissions, ZCode
tools, native skills, MCP, memory packets, GitNexus context, auto-merge,
approvals, repo mutation, or live `reviewConcurrency.maxActiveRuns`.

## Changes

- Add `buildReviewBudgetStatus` for deterministic active/queued capacity
  projection, including would-lease and delayed counts.
- Expose compact budget summaries in scheduler output, operator status, runtime
  inventory, and `release:status`.
- Add `budget-status` for detailed operator inspection with bounded row and
  detail caps.
- Keep queue visibility resilient by reporting budget read failures separately
  instead of failing the whole `queue` command.
- Preserve active leased/running rows before applying pending-row caps so
  capacity math remains accurate under large queue backlogs.
- Classify delayed queue work by actionable provider/org/repo/manual-reserve
  reasons before falling back to synthetic `lease_limit`.
- Document the new operator budget surfaces in `docs/operator-cli.md`.

## Runtime Caveats

- Budget projection is advisory operator telemetry; it does not lease, post, or
  skip PR heads by itself.
- `budget-status` fails closed when the scheduler is disabled or the pending
  queue input is truncated. `release:status` still includes compact budget
  telemetry without turning detail truncation into a release blocker.
- Pending-row caps apply only to queued/provider-deferred rows. Active
  leased/running rows are always included so active capacity counts remain
  correct.
