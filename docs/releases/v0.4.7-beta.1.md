## Purpose

Promote the merged NeonDiff Desktop MVP work and the self-repo exact-head
review gate hotfix as a single beta release. The release keeps the live worker
on tagged code while closing the agent incident where a final self-repo PR head
could be merged before evaOS review status settled.

## Changes

- Add the SwiftPM NeonDiff Desktop MVP scaffold for daemon status, monitored
  repo visibility, provider/model settings, license-key UI, command builders,
  config/status parsers, redaction, and a local smoke target.
- Add public config inspection/patch CLI plumbing used by the desktop MVP.
- Add `review-head-gate`, a read-only exact-head pre-merge gate for self-repo
  and release-critical PRs.
- Harden the gate so newer active re-review jobs block old processed proof,
  terminal posted queue evidence can pass when processed rows are absent,
  readiness-only passes require a review URL, and stale active queue residue
  cannot block a newer posted review forever.
- Document the gate boundary: it proves exact-head recorded non-blocking evaOS
  review state; GitHub App authorship remains verified by live App-backed
  `release-status` and launchd configuration.

## Runtime Notes

- This release does not widen live concurrency, monitored repos, posting
  policy, issue enrichment, native ZCode tools, or GitHub App permissions.
- The live default remains one active review run.
- The #182 incident root cause was not a ZCode quota problem. The bot reviewed
  earlier heads of #181, but the final head was pushed and merged before the
  open-PR poller recorded a terminal evaOS status for that exact SHA.
- `review-head-gate` is now the required self-repo/release-critical pre-merge
  check so future final-head races fail closed before merge.
