## Purpose

Promote the public confidence display guard for NeonDiff/evaOS review comments.
This release keeps model-supplied confidence as internal evidence only unless a
finding has calibrated evidence that meets the configured public-display
thresholds. Public comments continue to default to `uncalibrated`.

## Included Changes

- Adds a dedicated public confidence policy and sanitizer.
- Hides percentage-like confidence claims from public inline comments,
  walkthroughs, enrichment comments, and review-status surfaces unless
  calibration thresholds pass.
- Preserves internal confidence metadata for reviewer gating and future
  calibration.
- Adds truncation-safe sanitizer behavior so boundary fragments cannot leak
  confidence percentages.
- Extends focused tests for findings, enrichment, walkthroughs, review status,
  and public confidence policy.

## Caveats

- This release does not claim calibrated confidence accuracy.
- This release does not display public confidence percentages by default.
- This release does not close the broader confidence-calibration roadmap issue;
  it ships the public-display safety guard needed before later calibration work.
- This release does not change provider selection, GitNexus behavior, monitored
  repositories, issue-enrichment allowlists, or live concurrency settings.
