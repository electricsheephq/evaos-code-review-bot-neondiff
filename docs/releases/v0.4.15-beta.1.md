## Purpose

Promote the provider registry foundation for NeonDiff/evaOS review routing. This
release gives operators a safe way to inspect configured providers, validate
provider metadata, and run local-only smoke checks without expanding the live
reviewer's default execution mode beyond the existing ZCode-backed path.

## Included Changes

- Adds provider registry metadata for ZCode/GLM, OpenAI-compatible, OpenAI,
  Anthropic, Gemini, and Ollama-style providers.
- Adds provider operator commands for listing configured providers, explaining
  readiness, and running explicit smoke checks.
- Keeps hosted provider routes honest in documentation: prompts, diffs, and
  review context may leave the worker when a hosted provider is selected.
- Keeps no-egress use cases pointed at local providers such as ZCode or Ollama.
- Restricts OpenAI-compatible smoke checks to local loopback endpoints during
  this beta.
- Fails remote OpenAI-compatible smoke checks closed until the transport can pin
  the validated DNS result.
- Adds configuration defaults and examples for provider registry settings
  without changing the live default provider.
- Adds redacted evidence/output handling for provider operator surfaces.

## Caveats

- This release does not switch live PR reviews to an alternate provider.
- This release does not enable remote OpenAI-compatible smoke checks.
- This release does not add provider load balancing or automatic failover.
- This release does not claim provider quality parity, CodeRabbit parity, or
  calibrated review accuracy.
