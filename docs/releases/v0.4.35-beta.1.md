## Purpose

Promote the GitHub App install-scope proof hardening as a named source beta so
operators can distinguish real App visibility and pull-request permission from
fallback-token reads before checkout, provider execution, or review posting.

This release improves `doctor github` and GitHub API proof behavior. It does not
publish a new npm package, expand GitHub App permissions, change monitored repo
allowlists, or claim production license-service readiness.

## Included Changes

- Add structured GitHub App read proof fields for repository visibility,
  installation presence, metadata access, pull-request access, GitHub API status,
  and classified API errors.
- Make `doctor github` fail closed when App credentials are missing or when App
  installation-scope proof cannot read metadata and pull requests.
- Keep fallback-token reads useful for local development while preventing them
  from being reported as GitHub App installation proof.
- Scope manual redirect handling to install-scope probes so ordinary review
  reads still follow GitHub redirects.
- Cache installation tokens without re-resolving installation IDs on every read,
  while preserving installation-specific token safety.
- Document the expected `doctor github` fields and pre-checkout blocker behavior.
