## Purpose

Ship the scheduler/runtime half of issue enrichment without enabling live issue
comments. The daemon can now run a default-off issue-enrichment lane for repos in
the separate `issueEnrichment.allowlist`, record dry-run/deferred/skipped/posting
state in SQLite, and expose issue-enrichment runtime health separately from PR
queue health.

## Changes

- Add durable `issue_enrichment_records` state keyed by repo and issue number.
- Add `runIssueEnrichmentCycle` for issue enrichment scheduling, sticky issue
  comment upsert, duplicate suppression by unchanged `updated_at`, and
  redacted failed-post recording.
- Gate daemon issue-enrichment execution behind `issueEnrichment.enabled`.
- Fail closed before scanning or posting when live issue comments require App
  posting credentials that are unavailable.
- Add issue-enrichment runtime health to `status` and `runtime-inventory`,
  including disabled, idle, deferred, and error states.
- Keep runtime health totals independent from displayed row limits so older
  failed issue-enrichment records cannot be hidden by `--limit`.
- Preserve active live defaults: issue enrichment disabled, empty issue
  allowlist, and `postIssueComment=false`.
