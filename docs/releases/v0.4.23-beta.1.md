## Purpose

Expose provider/model provenance in the PR walkthrough so operators can tell
which configured review provider produced an App-authored review, without
claiming calibrated review accuracy or changing live provider selection.

## Included Changes

- Adds provider metadata to walkthrough comments.
- Adds redaction-aware formatting for provider metadata.
- Adds review settings preview coverage for provider metadata.
- Marks stale or missing provider registry entries as `Unregistered provider id`
  with `zcode (registry miss)` and model `unknown`.
- Keeps `unknown`, `dependency`, and `flaky_test_risk` P0/P1 findings eligible
  for `REQUEST_CHANGES`.
- Keeps `proof_gap` and `docs_only` P0/P1 findings advisory unless taxonomy
  inference validates a blocking risk category such as release regression.
- Preserves current-head inline review validation, secret redaction, duplicate
  suppression, and App-authored review behavior.

## Caveats

- PR `#236` was merged after CI and review-thread resolution while the rebased
  head was still waiting for evaOS current-head review. The worker correctly
  skipped the head after merge as `closed`.
- This release does not close `#108`; provider-parity fixture execution for
  GLM/Z.ai and one OpenAI-compatible local endpoint remains open.
- This release does not switch live reviews to an alternate provider.
- This release does not enable native ZCode skills, MCP, tools, memory, or
  target-repo mutation.
- Public comments intentionally include provider/model provenance for this MVP.
