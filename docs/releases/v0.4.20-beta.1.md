# v0.4.20-beta.1

## Outcome

Ships the public confidence calibration policy from #121. Public review comments
remain uncalibrated by default; confidence percentages are allowed only when a
calibration report proves mode, evidence URL, dataset id, labeled findings,
P0/P1 labels, negative controls, and Wilson lower-bound thresholds.

## Highlights

- Added `buildPublicConfidenceCalibrationReport` with explicit dataset, label,
  metric, request-changes, and proof-boundary fields.
- Added fail-closed policy evaluation for malformed confidence thresholds.
- Added `rejectedMinimum` and `blockedReason: malformed_minimum` so audit
  reports explain bad operator input without weakening the public display gate.
- Switched below-threshold codes to `*_below_required` so custom stricter
  minimums stay consistent with `metrics.*.required`.
- Added confidence calibration docs under `docs/evals/`.
