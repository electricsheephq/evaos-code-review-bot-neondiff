## Purpose

Add the public NeonDiff license-key client path required for private-repo and
update-channel entitlement checks, while preserving local-first behavior and
without deciding final hosted API or pricing terms in this code slice.

## Included Changes

- Adds `neondiff license activate/status/deactivate --json`.
- Stores license keys through the beta file backend with 0600 permissions by
  default; Keychain writes remain disabled until a native no-argv storage path
  exists.
- Writes a local entitlement cache containing status, expiry, repo visibility
  scope, update entitlement, plan metadata, and a short license fingerprint.
- Adds a private-repo worker gate that fails closed before worktree prep,
  provider/model calls, or GitHub review posting when license enforcement is
  enabled and no active private entitlement exists.
- Preserves dry-run behavior by skipping live license metadata lookups during
  review dry-runs.
- Clears cached entitlements after terminal API denial, rejects malformed 2xx
  API responses, treats internal repos as entitlement-required, and caps
  offline entitlement grace at 15 minutes.
- Keeps public repo reviews license-free when `license.publicReposFree` is true.
- Documents setup and example config for license activation.

## Caveats

- The hosted production license API is still not claimed healthy by the public
  manifest. This release adds the local client/gate and mockable contract.
- Live internal config keeps license enforcement disabled unless explicitly
  enabled in a future config promotion.
- This release does not publish packages, desktop artifacts, or a public
  download/license portal.
