## Purpose

Keep the live daemon from ever using the operator checkout as a review worktree,
then clear the release gate after two transient provider/network failures left
stale failed queue state behind.

## Changes

- Move the live `workRoot` outside the source checkout:
  `/Volumes/LEXAR/repos/evaos-code-review-bot-runtime`.
- Add protected-checkout root detection for process cwd, installed package root,
  and explicit `EVAOS_REVIEW_BOT_PROTECTED_CHECKOUT_ROOT` overrides.
- Reject review `workRoot`, mirror, and worktree paths that are inside a
  protected checkout or that contain a protected checkout.
- Harden path boundary checks so sibling names such as `..runtime` are not
  treated as parent traversal.
- Add symlink regression coverage for child `mirrors` and `worktrees`
  directories that resolve back into the source checkout.
- Document the packaged deployment requirement to set
  `EVAOS_REVIEW_BOT_PROTECTED_CHECKOUT_ROOT`.
