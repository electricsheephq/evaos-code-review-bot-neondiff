## Purpose

Promote the GitNexus post-merge refresh preflight so release operators do not
accidentally run `gitnexus analyze --embeddings` against an index whose current
embedding dimensions cannot be proven. The safe default is fail-closed for
embedding refreshes and an explicit `--index-only` fallback for commit freshness.

## Included Changes

- Adds `gitnexus-refresh-preflight` CLI support.
- Compares intended embedding provider/model/dimensions with explicit current
  GitNexus embedding-dimension evidence before recommending `--embeddings`.
- Requires current dimensions to come from explicit `embeddingDimensions` or
  `Embedding dimensions` fields.
- Rejects stale/noisy dimension text such as stray `7d`, bare `dimensions:`,
  and ambiguous dimension-change output.
- Preserves repo alias in recommended `gitnexus analyze` commands.
- Surfaces `gitnexus status` and `gitnexus doctor` subprocess failures as
  explicit preflight errors.
- Keeps live GitNexus context, review runtime, monitored repos, provider
  cooldowns, issue enrichment, and launchd config unchanged.

## Caveats

- This release does not run `gitnexus analyze --embeddings` unless the preflight
  proves explicit current embedding dimensions, except for the one-time
  post-fallback repair needed because local GitNexus discarded the existing
  vector cache during an index-only refresh.
- Local GitNexus `1.6.7-electric.2` does not expose the documented
  `--allow-dimension-change true` flag. Use explicit tracked migration evidence
  before changing dimensions instead of relying on that flag.
- This release does not change live GitNexus review context behavior or turn
  GitNexus on/off for running review agents.
- This release does not expand GitHub App permissions, issue-enrichment
  allowlists, monitored repositories, ZCode tools, native skills, MCP, or memory.
- `--allow-dimension-change true` remains out of routine release hygiene and is
  reserved for a separate tracked embedding migration issue.
