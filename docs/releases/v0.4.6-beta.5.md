## Purpose

Make the operator surfaces agree when provider-deferred queue state, stale run
leases, and failed durable jobs are what keep the live review bot red. This is
the observability and safe-cleanup patch needed before the remaining provider
throttle work can be debugged without guessing from stale comments.

## Changes

- Align `queue`, `provider-cooldowns`, `coverage-audit`, `budget-status`, and
  `release:status` around provider-deferred and stale-lease blockers.
- Add actionable queue rows, `runtimeOk`, failed gates, and recommended cleanup
  actions so operators can distinguish provider overload from daemon death.
- Add dry-run-first `clear-review-queue-leases` for stale run leases and
  orphaned active queue leases after launchd restarts.
- Fix command-specific help for nested commands such as `run-once --help`,
  `review-pr help`, and daemon help so they do not execute worker paths.
- Preserve scoped cleanup safety: repo/PR/job-filtered cleanup no longer deletes
  unrelated stale global run leases.
