## Purpose

Ship the first issue-enrichment implementation as a dry-run operator surface
only, then harden the activation-baseline audit path before re-adding any repo
that has a large backlog of old open pull requests.

This release also fixes the release-blocking state-store compatibility issue
found during full-suite validation after the activation-audit patch.

## Changes

- Add issue-enrichment dry-run support for operator inspection.
- Keep issue enrichment separate from PR monitoring:
  - no live issue polling;
  - no live issue comments;
  - no GitHub App Issues permission expansion;
  - no issue-enrichment allowlist activation.
- Track live issue enrichment rollout separately in `#135`, including the
  separate allowlist, per-repo throttles, and burst thresholds required before
  any active issue-enrichment rollout.
- Avoid historical PR verification sweeps for activation-baselined repos:
  - coverage audit skips live current-head re-reads for exact activation
    baseline rows;
  - command-comment lookup is skipped for activation-baselined historical heads;
  - scoped manual/trusted command paths can still review an activation-baselined
    head when explicitly targeted.
- Preserve normal safety reads for non-baseline processed heads so stale-list and
  new-head races still surface.
- Keep the worker baseline guard compatible with minimal state-store doubles
  used by existing focused tests.
