## Purpose

Promote the post-`v0.4.37-beta.1` source batch as a named local-worker beta
instead of leaving launchd on untagged `main`.

This release adds human-gated calibration promotion proof, public setup and
operator documentation hardening, hermetic QA-lab timing fixtures, and the first
license-service admin status contract slice. It does not publish a new npm
package, enable hosted license billing, widen runtime permissions, expand
monitored repos, or change the public install package from
`neondiff@0.4.30-beta.1`.

## Included Changes

- Add human-gated calibration promotion plumbing with explicit negative-control
  and category-precision floor checks before any public confidence promotion.
- Refresh the README, operator CLI docs, setup validation docs, vision doc, and
  issue-enrichment guidance for the current public beta shape.
- Neutralize example config credential values and document environment-based
  provider credential reads.
- Add `neondiff --version` and per-command help surface coverage for public CLI
  usability.
- Add a hermetic QA-lab timing harness and seeded scenario fixtures for
  repeatable local evaluation without contacting providers.
- Add the license-service admin status response contract, entitlement metadata
  handling, private-repo gating proof, cache redaction hardening, and
  keychain-cache fallback tests.
- Fill the public release manifest source SHA state for the promoted release
  line while keeping the package artifact held at `neondiff@0.4.30-beta.1`.

## Caveats

- This release does not publish a new npm package; the public npm package
  remains `neondiff@0.4.30-beta.1`.
- This release does not implement or deploy the production hosted license
  service, billing lifecycle, customer portal, or license-service admin API.
- License-service admin status data remains a source/admin contract slice and
  local client proof, not hosted production readiness.
- Calibration promotion gates are plumbing and proof surfaces; this release does
  not claim CodeRabbit parity, calibrated 95% confidence, or autonomous human
  review replacement.
- GitNexus remains advisory/read-only where enabled; this release does not
  expand GitNexus runtime permissions or target-repo mutation.
- This release does not change monitored repos, GitHub App permissions, native
  ZCode skills, MCP tools, memory, shell tools, auto-approval behavior, or
  target-repo mutation.
