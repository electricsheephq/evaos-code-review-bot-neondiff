## Purpose

Promote the public `neondiff` / `evaos-review-bot` source-checkout CLI flow
while preserving the live bot's existing safety boundary. Live one-off
`review-pr` posting now requires both an explicit config file and an approved
PR head SHA before it can post.

## Changes

- Add the packaged public CLI surface and docs for source-checkout operators.
- Require `review-pr --dry-run false` to include `--confirm true`.
- Require live `review-pr` to include an existing `--config` path instead of
  silently falling back to the default config.
- Require live `review-pr` to include an approved head SHA via `--head-sha` or
  `--expected-head`.
- Reject conflicting `--head-sha` / `--expected-head` values before config load
  or review execution.
- Compare the freshly fetched PR head against the approved head in the worker
  before any posting path can continue.
- Repair a release-gate test fixture whose hard-coded lease timestamp expired
  on 2026-07-03.
