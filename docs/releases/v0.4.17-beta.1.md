## Purpose

Promote the review queue burst/backpressure fix for live beta operation. This
release is aimed at preventing bursty repositories from saturating the durable
review queue and leaving agents waiting on `queued` status without a clear
capacity reason.

## Included Changes

- Adds repo-profile scheduler controls for `maxActiveHeads`, `maxQueuedHeads`,
  and `overflowAction`.
- Keeps default overflow behavior as `defer`, with current-head readiness and
  status-comment evidence instead of duplicate status churn.
- Documents `overflowAction: "skip"` as a terminal exact-head policy; transient
  burst recovery should use the default `defer`.
- Applies repo-profile active caps during the same cycle that burst jobs are
  freshly enqueued.
- Aligns coverage/operator surfaces so capacity-deferred heads appear as
  provider-deferred work without requiring a durable queue row.
- Adds durable queue oldest-waiting diagnostics, including malformed timestamp
  visibility for operator dashboards.
- Keeps GitNexus context behavior unchanged; the focused GitNexus context test
  remains in the release gate.
