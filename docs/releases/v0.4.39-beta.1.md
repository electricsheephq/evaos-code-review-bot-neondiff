## Purpose

Promote the `#361` CodeQL remediation as a named source beta instead of
leaving the live worker on untagged `main`.

This release routes review/evidence artifact writes through a shared secure
helper. The helper writes to a private same-directory temp file with owner-only
permissions, then atomically renames it over the target so retries or reused
evidence directories cannot preserve looser file modes from older artifacts.

## Included Changes

- Add `writeSecureFileSync` for local review/evidence artifact writes.
- Migrate redacted evidence writes in enrichment, walkthrough posting, worker
  evidence, and ZCode stdout/stderr capture paths to the secure helper.
- Preserve existing redaction/content behavior while tightening filesystem
  permissions for newly created and overwritten evidence artifacts.
- Add focused temp-file tests for new-file and pre-existing-file overwrite
  paths, including cleanup of private temp sidecars.
- Resolve the `#359` security gate that kept 10 main-branch
  `js/insecure-temporary-file` alerts open.
