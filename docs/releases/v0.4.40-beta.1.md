## Purpose

Promote the post-`v0.4.39-beta.1` source batch as a named local-worker beta
instead of letting the live worker drift onto untagged `main`.

This release gathers the current-main worker and productization improvements
that landed after the secure evidence-write remediation: risk-weighted queue
aging, review mode routing, per-head public command policy, weighted related
context, lifecycle/fixer markers, scheduled outcome observation, desktop
credential preflight proof, and license-service scaffolding.

## Included Changes

- Add a lease-time aging guarantee and starvation simulation for risk-weighted
  queue behavior so required-validation surfaces do not permanently starve
  docs-only work.
- Add a focused category-precision floor gate test for review-gate calibration
  hardening.
- Add the license service activate/validate/deactivate API, admin issuance CLI,
  Fly.io deploy assets, and deployment runbook as source/admin scaffolding.
- Add public review and re-review command policy with per-head cooldowns.
- Add weighted related-context relevance scoring for issue/PR enrichment
  references.
- Add desktop credential preflight doctor coverage and CI secret-naming policy
  documentation for future desktop release work.
- Add review mode routing for depth/spend routing and config-promoted precision
  signals.
- Add daemon-integrated scheduled outcome observation for revert, hotfix,
  merged-fix, and human-thread signals.
- Add lifecycle markers and fixer-handoff metadata for review-status comments.

## Caveats

- This release does not publish a new npm package; the public npm package
  remains `neondiff@0.4.30-beta.1`.
- The license service remains source/admin scaffolding and runbook proof, not a
  hosted production license-service rollout.
- Desktop credential work is preflight proof only; this release does not ship a
  signed or notarized desktop artifact.
- Scheduled outcome observation and lifecycle markers are runtime-safety and
  evidence improvements; this release does not claim CodeRabbit parity,
  calibrated 95% confidence, GA readiness, or human-review replacement.
- This release does not change monitored repos, GitHub App permissions, live
  review concurrency, issue-enrichment allowlists, provider selection, model
  selection, native ZCode skills, MCP tools, memory, shell tools, auto-approval
  behavior, or target-repo mutation.
