## Purpose

Prepare the live PR reviewer to safely re-add
`Martian-Engineering/lossless-claw` without sweeping its historical open PR
backlog. The release keeps `lossless-claw` excluded from the live 19-repo PR
monitor until the post-release temp-config proof shows that only genuinely new
eligible heads are unprocessed or queued.

## Changes

- Add an explicit activation-policy helper for new-only repo activation.
- Require evidence that a pre-activation PR was actually baselined before
  suppressing a later changed head as `activation_baseline_existing_head`.
- Record activation-baseline skip rows from both scheduler and worker paths so
  coverage and operator status can explain skipped historical heads instead of
  treating them as invisible.
- Check active queue jobs before classifying a head as activation-baseline
  skipped, so trusted scoped command work remains visible as queued.
- Retire superseded queued heads before early activation-baseline returns.
- Normalize activation-baseline readiness reasons so dashboard and evidence
  telemetry use one reason bucket.
- Preserve the trusted scoped command escape hatch: explicit commands can still
  request review of pre-activation PR heads.
