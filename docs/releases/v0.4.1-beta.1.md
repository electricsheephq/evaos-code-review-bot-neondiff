## Purpose

Add default-off read-only skill-pack prompt context and sticky PR enrichment
plumbing without widening live permissions or changing the bot's review-only
runtime posture.

This closes the read-only skill-pack context work in `#7` and lands the PR
enrichment MVP for `#10`. Full issue enrichment, new/changed issue polling, and
stale-issue handling remain open `#10` follow-up scope.

## Changes

- Add `skillPacks` config with default-off activation, allowlisted files,
  byte/token caps, realpath root containment, symlink escape rejection, unsafe
  directive rejection, redaction, provenance, and hard rendered-packet bounds.
- Add `build-skill-pack` operator CLI evidence command.
- Add ZCode prompt integration for read-only skill-pack packets while keeping
  native ZCode skills, MCP, tools, shell, web, memory, agents, and writes
  disabled.
- Add `enrichment` config with default-off activation and optional sticky PR
  issue-comment posting behind App credentials and non-dry-run mode.
- Add `build-enrichment-comment` operator CLI evidence command for PR metadata.
- Add sticky PR enrichment markers, head-specific rendered-body state hashes,
  suggestion-only labels/reviewers, validation suggestions, triage-gap hints,
  and redacted upsert error evidence.
- Stabilize `daemon-loop` heartbeat testing by stubbing the provider-cooldown
  retry lane in the successful-cycle unit test.

  App-authored issue-comment upsert path, but live posting remains disabled in
  the active config.
