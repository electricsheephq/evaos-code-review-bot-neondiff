## Purpose

Complete the `v0.4.5` issue-enrichment safety release by fixing the operator
retirement path used during release health gates. `v0.4.5-beta.1` shipped the
runtime code, but `release-status` remained red because `retire-failed` retired
the processed review row without retiring the matching failed durable queue job.

## Changes

- Make `retire-failed` retire matching failed durable queue jobs as
  `stale_retired`.
- Preserve the original retired processed-review error in the queue-job
  `last_error`.
- Make `retire-failed` idempotent for already-retired processed rows so a
  partial old-command retirement can be completed without raw SQLite edits.
